# 의존성 규칙

모듈 간 의존성은 레이어 방향성을 따릅니다.
이 규칙은 구조 복잡도를 줄이고, 빌드 순환 의존성을 예방하기 위한 규칙입니다.

## Layer

### App
- 실제 실행 진입점 (iOS App Target)
- Feature를 조합하여 화면과 플로우를 구성

### Feature
- 사용자 기능 단위(UI + 상태 + 흐름) 모듈
- 도메인 요구사항 변화에 가장 민감
- Domain을 통해 비즈니스 로직을 사용

### Domain
- 도메인 관점의 핵심 비즈니스 규칙, DTO 정의
- Feature에서 Interface 형태로 의존

### Core
- 앱 전반에서 공통으로 사용하는 기반 기능
- 네트워크, 네이티브 프레임워크 등

### Shared
- 모든 레이어에서 사용 가능한 최소 단위 모듈
- 타입, 확장, 상수 등 순수 공용 코드
- 다른 레이어에 의존하지 않음

## 기본 방향
![](layer)

- Feature는 Domain, Core, Shared에 의존할 수 있습니다.
- Domain은 Core, Shared에 의존할 수 있습니다.
- Core는 Shared에 의존할 수 있습니다.
- Shared는 어느 레이어에도 의존하지 않습니다.

## 인터페이스 타깃 사용

공개 API와 구현을 분리해야 하는 경우 `Interface` 타깃을 사용합니다.
상위 레이어는 구현 타깃 대신 인터페이스 타깃에 의존하도록 구성합니다.
